Core Features

六大核心功能模块

覆盖预约管理全流程，从规则配置到数据统计，提供完整的业务闭环能力。

C端小程序预约
用户通过微信小程序完成项目浏览、预约表单填写、须知确认、时段选择，支持 10+ 种校验场景实时反馈。
灵活规则配置
接待量上限、提前天数、取消截止、陪同人数限制、重复预约检测等 6+ 条可配置业务规则。
资格验证体系
姓名 + 人才编号双重匹配验证，可导入人员名单，支持模板批量导入与单条新增，确保预约质量。
扫码签到签退
Canvas 动态生成二维码（预约码 / 签退码双码区分），支持扫码签到与手动确认两种方式。
自定义字段引擎
预置字段 + 自定义字段扩展（文本/数字/日期/下拉），支持拖拽排序、必填开关、启用控制。
记录管理与统计
完整预约记录列表，支持搜索筛选导出，统计卡片展示总预约/今日/待到访/已取消等关键指标。

Business Flow

核心业务流程

从管理员配置规则到用户完成签到签退，四步完成一次完整的预约闭环。

1
配置规则
管理员在 PC 后台配置预约规则：时段、接待量、人员名单、字段设置等
2
用户预约
C端用户扫码进入小程序 → 填写表单 → 系统自动校验规则 → 提交成功
3
签到签退
用户到访出示预约码 → 管理员扫码确认签到 → 接待完成后操作签退
4
数据管理
管理员在后台查看所有预约记录、统计数据，支持搜索筛选与导出
Technical Specs

技术规格一览

基于现代 Web 技术栈构建的原型系统，具备完整的交互能力和视觉表现力。

C端小程序规格
  • 设计语言：「琥珀光」暖铜金配色 (#d4855e)，圆角卡片式布局
  • 页面数量：首页 + 表单页 + 成功页 + 我的预约 + 详情页 = 5个主页面
  • 校验场景：4种Toast轻提示 + 6种阻断式弹窗 = 共10种校验失败场景
  • 交互组件：横向日期条滑动、时段网格选择、Toast居中弹窗、Modal阻断弹窗
  • 状态流转：待到访 → 已签到 → 已签退；待到访 → 已取消
PC端后台规格
  • 设计语言：品牌橙 (#f05a28)，左侧固定导航 + 右侧内容区经典布局
  • 功能模块：预约设置(5Tab) + 预约记录 + 统计卡片 = 共7大功能区
  • 规则配置：时段接待量、提前天数上下限、取消截止、陪同人数、重复检测共6条
  • 特色功能：Canvas手绘二维码、文档区侧边抽屉、紧凑表格字段编辑
  • 弹窗组件：二维码×2 + 详情 + 取消确认 + 新增人员 + 导入 + 新增字段 = 7类弹窗
